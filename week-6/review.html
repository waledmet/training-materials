The Problem Without Databases

# Without database
posts = [
    {"title": "Post 1", "content": "Hello"},
    {"title": "Post 2", "content": "World"}
]

When server restarts... 💥 All data GONE!

The Solution: Databases

# With database
Post.objects.all()
  • ✅ Data persists forever
  • ✅ Handle millions of records
  • ✅ Fast searching & filtering
  • ✅ Multiple users, no conflicts

Think of it like a Filing Cabinet

Database = Filing Cabinet
Table = Drawer
Row = Folder
Column = Field in Document

Each drawer (table) holds related information

Django Model = Python Class

class Book(models.Model):
    title = models.CharField(max_length=200)
    author = models.CharField(max_length=100)
    pages = models.IntegerField()

This creates a database table!

Common Field Types

# Text
CharField(max_length=200)    # Short text
TextField()                   # Long text

# Numbers
IntegerField()               # Whole numbers
DecimalField()               # Precise decimals

# Date/Time
DateTimeField()              # Date and time
DateField()                  # Date only

# Boolean
BooleanField()               # True/False

Auto Timestamps

class Post(models.Model):
    # Set once when created
    created_at = models.DateTimeField(auto_now_add=True)

    # Update every save
    updated_at = models.DateTimeField(auto_now=True)

Django handles these automatically!

Making Models Readable

class Book(models.Model):
    title = models.CharField(max_length=200)
    author = models.CharField(max_length=100)

    def __str__(self):
        return self.title

Without __str__: "Book object (1)"
With __str__: "The Great Gatsby"

What Are Migrations?

Version control for your database

1. Change Model → 2. makemigrations → 3. migrate → 4. Database Updated

Think of migrations as moving instructions for your database

Two-Step Process

# Step 1: Create migration (plan)
python manage.py makemigrations

# Step 2: Apply migration (execute)
python manage.py migrate

Always in this order!

What is ORM?

Object-Relational Mapping

Write Python, not SQL!

# Instead of SQL:
SELECT * FROM books WHERE pages > 300;

# Write Python:
Book.objects.filter(pages__gt=300)

Create, Read, Update, Delete

# Create
Book.objects.create(title="Django Book", pages=400)

# Read
books = Book.objects.all()
book = Book.objects.get(id=1)

# Update
book.title = "New Title"
book.save()

# Delete
book.delete()

QuerySet Methods

# All records
Book.objects.all()

# Filter (multiple results)
Book.objects.filter(pages__gt=300)

# Get (single result)
Book.objects.get(id=1)

# Exclude
Book.objects.exclude(pages__lt=100)

# Order
Book.objects.order_by('-pages')

Powerful Filtering

# Greater than
Book.objects.filter(pages__gt=300)

# Contains (case-insensitive)
Book.objects.filter(title__icontains="django")

# Starts with
Book.objects.filter(title__startswith="The")

# Range
Book.objects.filter(pages__range=(100, 300))

Double underscores are the magic!

Make Admin Powerful

@admin.register(Book)
class BookAdmin(admin.ModelAdmin):
    list_display = ('title', 'author', 'pages')
    list_filter = ('author',)
    search_fields = ('title', 'author')
    ordering = ('-pages',)

Now you have search, filters, and sorting!

ForeignKey - Many to One

class Author(models.Model):
    name = models.CharField(max_length=100)

class Book(models.Model):
    title = models.CharField(max_length=200)
    author = models.ForeignKey(Author, on_delete=models.CASCADE)

Many books → One author

Forward and Backward

# Forward (book → author)
book = Book.objects.get(id=1)
print(book.author.name)

# Backward (author → books)
author = Author.objects.get(id=1)
for book in author.books.all():
    print(book.title)

# Filter by related
books = Book.objects.filter(author__name="Austen")

Many to Many Relationships

class Student(models.Model):
    name = models.CharField(max_length=100)
    courses = models.ManyToManyField('Course')

class Course(models.Model):
    name = models.CharField(max_length=100)

Many students ↔ Many courses

Model Best Practices

  • ✅ Always add __str__() method
  • ✅ Use descriptive field names
  • ✅ Add related_name to relationships
  • ✅ Use blank=True for optional fields
  • ✅ Add Meta class for ordering
  • ✅ Keep models focused and simple
