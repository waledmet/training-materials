## بطاقات مرجعية سريعة

### وسوم HTML الأساسية
```html
<!-- Structure -->
<!DOCTYPE html>, <html>, <head>, <body>

<!-- Content -->
<h1>-<h6>, <p>, <div>, <span>

<!-- Links & Images -->
<a href="...">, <img src="..." alt="...">

<!-- Lists -->
<ul>, <ol>, <li>

<!-- Forms -->
<form>, <input>, <textarea>, <button>, <select>

<!-- Semantic -->
<header>, <nav>, <main>, <section>, <footer>
```

### خصائص CSS الأساسية
```css
/* Text */
color, font-size, font-family, font-weight, text-align

/* Box Model */
width, height, padding, margin, border

/* Background */
background-color, background-image

/* Display */
display, position, float

/* Flexbox */
display: flex, justify-content, align-items
```

### أوامر Git الأساسية
```bash
git init          # Initialize repository
git status        # Check status
git add .         # Stage all files
git commit -m ""  # Commit with message
git log           # View history
git push          # Push to remote
git pull          # Pull from remote
```

## نصائح لاستخدام الموارد

1. **لا تحاول تعلم كل شيء**: ركّز على مواضيع الأسبوع الثالث أولًا
2. **استخدم مصادر متعددة**: اختلاف الشرح يساعد
3. **ضع إشارات مرجعية**: ابنِ مكتبة مواردك
4. **الممارسة أساسية**: القراءة وحدها لا تكفي
5. **انضم للمجتمعات**: اسأل وساعد الآخرين
6. **ابنِ مشاريع**: أفضل طريقة للتعلم
7. **احتفظ بالتوثيق قريبًا**: MDN صديقك

## الحصول على المساعدة

### عندما تتعثر

1. **اقرأ رسائل الخطأ**: غالبًا تخبرك بالمشكلة
2. **ابحث في Google**: "how to [what you want] html/css"
3. **افتح MDN**: المصدر الأكثر موثوقية
4. **اسأل في Stack Overflow**: اكتب السؤال بشكل جيد
5. **استخدم DevTools**: افحص وتتبع الأخطاء
6. **اسأل المدرب**: موجود للمساعدة

### كيف تسأل سؤالًا جيدًا

1. **كن محددًا**: "شريط التنقل غير متمركز" بدلًا من "لا يعمل"
2. **أرفق الكود**: استخدم CodePen أو الصق الكود بتنسيق جيد
3. **اشرح ما جربته**: اذكر محاولاتك
4. **قدّم السياق**: المتصفح، النظام، التفاصيل المهمة
