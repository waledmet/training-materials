Lists - Introduction

What is a List?

  • Ordered collection of items
  • Can contain any data type
  • Items can be mixed types
  • Mutable (can change after creation)
  • Each item has an index (position)
Shopping List:
[0]     [1]      [2]     [3]
["milk", "eggs", "bread", "butter"]

Creating Lists:

# Empty list
shopping = []

# List with items

Lists - Accessing Elements

Index Position

fruits = ["apple", "banana", "cherry"]
          0       1         2         (forward index)
         -3      -2        -1        (backward index)

Accessing Items:

fruits = ["apple", "banana", "cherry"]

# Forward indexing (0-based)
print(fruits[0])    # "apple"
print(fruits[1])    # "banana"
print(fruits[2])    # "cherry"

# Negative indexing (from end)
print(fruits[-1])   # "cherry"
print(fruits[-2])   # "banana"

Common Error:

print(fruits[5])    # IndexError: list index out of range

Lists - Slicing

Get Multiple Elements

fruits = ["apple", "banana", "cherry", "date", "elderberry"]

# Get elements from index 1 to 3 (3 NOT included)
print(fruits[1:3])      # ["banana", "cherry"]

# Get first 3 elements
print(fruits[:3])       # ["apple", "banana", "cherry"]

# Get last 2 elements
print(fruits[-2:])      # ["date", "elderberry"]

# Get with step
print(fruits[::2])      # ["apple", "cherry", "elderberry"]

# Reverse list
print(fruits[::-1])     # ["elderberry", "date", "cherry", "banana", "apple"]

Syntax: list[start:stop:step]

  • start: where to begin (default 0)
  • stop: where to end (NOT included)
  • step: jump size (default 1)

Lists - Modifying Elements

Changing Items in a List

# Change multiple items (slice assignment)

Lists - Removing Elements

Delete Items

fruits = ["apple", "banana", "cherry", "date"]

# Remove by index
fruits.pop(1)      # Removes "banana", returns it
print(fruits)      # ["apple", "cherry", "date"]

Tuples - Introduction

Immutable Lists

What is a Tuple?

  • Like a list but CANNOT be changed after creation
  • Immutable (fixed, unchangeable)
  • Slightly faster and use less memory than lists
  • Can be used as dictionary keys (lists cannot)

Creating Tuples:

# Empty tuple
empty = ()

# Single item tuple (note the comma!)
single = (42,)

# Multiple items
coordinates = (10, 20)
rgb = (255, 128, 0)
person = ("Ahmed", 25, "Cairo")

Accessing Items (same as lists):

person = ("Ahmed", 25, "Cairo")
print(person[0])      # "Ahmed"
print(person[-1])     # "Cairo"
print(person[1:3])    # (25, "Cairo")
Trainer Note: Emphasize immutability - explain WHY we'd want immutable data

Tuples - Properties

When to Use Tuples

# Use as dictionary keys (can't use lists)
locations = {
    (10, 20): "Building A",
    (30, 40): "Building B",
    (50, 60): "Building C"
}
print(locations[(10, 20)])  # "Building A"

# Unpacking values
person = ("Ahmed", 25, "Cairo")
name, age, city = person
print(f"{name} is {age} years old in {city}")

# Return multiple values from function
def get_coordinates():
    return (10, 20)

x, y = get_coordinates()
print(f"X: {x}, Y: {y}")

Tuple Methods (limited):

t = (1, 2, 3, 2, 1)
print(t.count(2))     # 2
print(t.index(3))     # 2

Dictionaries - Introduction

Key-Value Pairs

What is a Dictionary?

  • Collection of key-value pairs
  • Like a real dictionary (word → definition)
  • Unordered (order doesn't matter)
  • Mutable (can change)
  • Each key is unique
  • Keys must be immutable (strings, numbers, tuples)

Creating Dictionaries:

# Empty dictionary
empty = {}

# With items
student = {
    "name": "Ahmed",
    "age": 25,
    "city": "Cairo",
    "gpa": 3.8
}

# Alternative syntax
person = dict(name="Fatima", age=30, city="Riyadh")

Dictionaries - Accessing & Modifying

Work with Key-Value Pairs

student = {
    "name": "Ahmed",
    "age": 25,
    "city": "Cairo",
    "gpa": 3.8
}

# Access values using keys
print(student["name"])      # "Ahmed"
print(student["age"])       # 25

# Safe access (won't error if key missing)
print(student.get("name"))  # "Ahmed"
print(student.get("email"))     # None (safe!)
print(student.get("email", "N/A"))  # "N/A" (with default)

# Add new key-value pair
student["email"] = "ahmed@university.edu"

# Modify existing value
student["age"] = 26

# Check if key exists
if "name" in student:
    print("Name exists")

# Delete a key-value pair
del student["email"]

# Get all keys, values, items
print(student.keys())       # dict_keys(['name', 'age', 'city', 'gpa'])
print(student.values())     # dict_values(['Ahmed', 26, 'Cairo', 3.8])
print(student.items())      # dict_items([('name', 'Ahmed'), ...])

Dictionaries - Iterating

Loop Through Dictionary

student = {
    "name": "Ahmed",
    "age": 25,
    "city": "Cairo",
    "gpa": 3.8
}

# Iterate over keys
for key in student:
    print(key)  # Prints: name, age, city, gpa

# Iterate over keys (explicit)
for key in student.keys():
    print(key)

# Iterate over values
for value in student.values():
    print(value)  # Prints: Ahmed, 25, Cairo, 3.8

# Iterate over both (best practice)
for key, value in student.items():
    print(f"{key}: {value}")
    # Output:
    # name: Ahmed
    # age: 25
    # city: Cairo
    # gpa: 3.8

# Count grades
grades = {"Ahmed": 85, "Fatima": 92, "Ali": 78}
for student_name, grade in grades.items():
    print(f"{student_name} scored {grade}")

Sets - Introduction

Unique Values Only

What is a Set?

  • Unordered collection of unique items
  • No duplicates allowed (automatically removes them)
  • Mutable but items must be immutable
  • Fast membership checking (x in set)
  • Mathematical set operations (union, intersection)

Creating Sets:

# Empty set (use set(), NOT {})
empty = set()

# Set with items
colors = {"red", "green", "blue"}
numbers = {1, 2, 3, 4, 5}

# Set removes duplicates automatically
items = {1, 2, 2, 3, 3, 3, 4}
print(items)  # {1, 2, 3, 4}

# From list (removes duplicates)
numbers_list = [1, 1, 2, 2, 3, 4, 4, 5]
unique = set(numbers_list)
print(unique)  # {1, 2, 3, 4, 5}
Trainer Note: Explain that order is NOT preserved in sets

Sets - Operations

Mathematical Set Operations

# Create sets
fruits1 = {"apple", "banana", "cherry", "date"}
fruits2 = {"banana", "date", "elderberry", "fig"}

# Union (all items from both)
all_fruits = fruits1 | fruits2
print(all_fruits)
# {"apple", "banana", "cherry", "date", "elderberry", "fig"}

# Intersection (items in both)
common = fruits1 & fruits2
print(common)  # {"banana", "date"}

# Difference (items in first but not second)
only_in_first = fruits1 - fruits2
print(only_in_first)  # {"apple", "cherry"}

# Symmetric difference (items in either but not both)
unique_to_each = fruits1 ^ fruits2
print(unique_to_each)  # {"apple", "cherry", "elderberry", "fig"}

# Check subsets and supersets
small = {1, 2}
large = {1, 2, 3, 4}
print(small.issubset(large))      # True
print(large.issuperset(small))    # True
print(small.isdisjoint({5, 6}))   # True (no common elements)

Sets - Methods

Adding and Removing

colors = {"red", "green", "blue"}

# Add single item
colors.add("yellow")
print(colors)  # {"red", "green", "blue", "yellow"}

# Add won't duplicate
colors.add("red")
print(colors)  # Still {"red", "green", "blue", "yellow"}

# Remove item (error if not exists)

# Discard (safe remove - no error if missing)
colors.discard("purple")  # No error!

# Pop removes arbitrary item

# Clear all items
colors.clear()  # Set is now empty

Day 2 - Functions

Why Functions?

Without Functions (repetitive):

# Calculate area - Method 1
length = 5
width = 3
area = length * width
print(f"Area: {area}")

# Calculate area - Method 2
length = 10
width = 4
area = length * width
print(f"Area: {area}")

# ... repeating same code

With Functions (clean):

def calculate_area(length, width):
    return length * width

print(calculate_area(5, 3))  # 15
print(calculate_area(10, 4)) # 40
print(calculate_area(7, 2))  # 14

Benefits:

  • Don't Repeat Yourself (DRY principle)
  • Reusable code
  • Easier to maintain and test

Functions - *args and **kwargs

Variable Arguments

*args - Variable Number of Arguments:

def add(*args):
    total = 0
    for num in args:
        total += num
    return total

print(add(5, 3, 2))           # 10
print(add(5, 3, 2, 1, 4))     # 15

    for key, value in kwargs.items():

# Output:
# name: Ahmed
# age: 25
# city: Cairo

# Real example
def create_user(**kwargs):
    return kwargs

print(user)  # {'name': 'Ahmed', 'email': 'ahmed@email.com', 'role': 'admin'}

Functions - Scope

Variable Scope - Where Variables Exist

# Global scope
x = 10  # Global variable

def outer():
    y = 20  # Local to outer()

    def inner():
        z = 30  # Local to inner()
        print(x)  # Can access global
        print(y)  # Can access from enclosing scope
        print(z)  # Can access own local

    inner()
    # print(z)  # ERROR! z doesn't exist here

outer()
# print(y)  # ERROR! y doesn't exist here

Best Practice - Avoid global variables:

# ✗ Bad - uses global
counter = 0
def increment():
    global counter
    counter += 1

# ✓ Good - uses parameter and return
def increment(count):
    return count + 1

counter = 0
counter = increment(counter)  # counter is now 1

Day 3 - Modules & Packages

Organizing Code

What is a Module?

  • A Python file containing code
  • Can contain variables, functions, classes
  • Extension is .py

What is a Package?

  • A directory containing Python modules
  • Must have __init__.py file
  • Like a folder of related modules

Real Example - Standard Library:

import math              # Module

Importing Modules

Different Import Styles

# Style 1: Import entire module
import math
print(math.sqrt(9))     # 3.0
print(math.pi)          # 3.141592653589793

# Style 2: Import specific item
from math import sqrt, pi
print(sqrt(9))          # 3.0
print(pi)               # 3.141592653589793

from math import sqrt as square_root

Error Handling - Why It Matters

Programs Break - Handle Gracefully

Without Error Handling:

user_input = input("Enter a number: ")
result = 100 / int(user_input)
print(f"Result: {result}")

# If user enters "abc", program CRASHES:
# ValueError: invalid literal for int()

With Error Handling:

try:
    user_input = input("Enter a number: ")
    result = 100 / int(user_input)
    print(f"Result: {result}")
except ValueError:
    print("Error: Please enter a valid number!")
except ZeroDivisionError:
    print("Error: Cannot divide by zero!")

Why Handle Errors?

  • Users make mistakes (enter wrong type)
  • Files might not exist
  • Network connections fail
  • Graceful failure beats crashes!
Trainer Note: Emphasize user experience - don't leave them hanging!

Try-Except Blocks

Basic Syntax

try:
    # Code that might cause an error
    risky_code()
except SomeError:
    # What to do if that error happens
    handle_error()

Common Errors:

# ValueError - wrong type
try:
    age = int(input("Enter age: "))
except ValueError:
    print("Age must be a number!")

# ZeroDivisionError - divide by zero
try:
    result = 100 / 0
except ZeroDivisionError:
    print("Cannot divide by zero!")

# IndexError - list index out of range
try:
    items = [1, 2, 3]
    print(items[10])
except IndexError:
    print("Item not found in list")

# KeyError - dictionary key missing
try:
    student = {"name": "Ahmed", "age": 25}
    print(student["email"])
except KeyError:
    print("Email not in student record")
Trainer Note: Show how to look up exceptions in Python docs

Multiple Except Blocks

Handle Different Errors

def calculator():
    try:
        num1 = float(input("First number: "))
        num2 = float(input("Second number: "))
        operation = input("Operation (+, -, *, /): ")

        if operation == "+":
            result = num1 + num2
        elif operation == "-":
            result = num1 - num2
        elif operation == "*":
            result = num1 * num2
        elif operation == "/":
            result = num1 / num2
        else:
            print("Unknown operation")
            return

        print(f"Result: {result}")

    except ValueError:
        print("Error: Please enter valid numbers")
    except ZeroDivisionError:
        print("Error: Cannot divide by zero")
    except Exception as e:
        print(f"Unexpected error: {e}")

calculator()

Finally Block (runs regardless):

try:
    file = open("data.txt")
    data = file.read()
except FileNotFoundError:
    print("File not found")
finally:
    file.close()  # Always runs, even if error

Day 4 - Virtual Environments

Keep Projects Isolated

What is a Virtual Environment?

  • Isolated Python environment per project
  • Separate packages per project
  • No conflicts between projects
  • Like a sandbox for your project

Why Use Virtual Environments?

Project A needs: Django 3.0, Flask 1.1.2
Project B needs: Django 4.0, Flask 2.0.0

Without Virtual Environments:
- Can only install one version of each
- Projects will break!

With Virtual Environments:
- Project A has its own Python + packages
- Project B has its own Python + packages
- No conflicts!
Trainer Note: Explain this prevents "dependency hell"

Creating Virtual Environments

Step by Step

Windows:

# Create virtual environment
python -m venv venv

# Activate it
venv\Scripts\activate

# You'll see: (venv) before your command line
# Install packages (only in this venv)
pip install django flask requests

# Deactivate when done
deactivate

Mac/Linux:

# Create virtual environment
python3 -m venv venv

# Activate it
source venv/bin/activate

# You'll see: (venv) before your command line
# Install packages

# Deactivate when done
deactivate

Project - Main Application

User Interface & Menu

"""To-Do List Application - Main Module"""

import json
import os

tasks = []
task_id_counter = 1

def display_menu():
    """Show main menu"""
    print("\n" + "="*40)
    print("        TO-DO LIST APPLICATION")
    print("="*40)
    print("1. Add new task")
    print("2. View all tasks")
    print("3. Mark task as complete")
    print("4. Delete task")
    print("5. Save and exit")
    print("="*40)

def display_tasks():
    """Display all tasks nicely formatted"""
    if not tasks:
        print("\nNo tasks yet! Add one to get started.")
        return

    print("\n📋 YOUR TASKS:")
    print("-" * 50)

    for task in tasks:
        status = "✓" if task["completed"] else "○"
        print(f"{status} [{task['id']}] {task['title']}")

    print("-" * 50)
    completed = len([t for t in tasks if t["completed"]])
    print(f"Total: {len(tasks)} | Completed: {completed}")

Project - File Operations

Save and Load Tasks

def save_tasks():
    """Save tasks to file"""
    try:
        with open("tasks.json", "w") as f:
            json.dump(tasks, f, indent=2)
        print("✓ Tasks saved successfully!")
    except Exception as e:
        print(f"Error saving tasks: {e}")

def load_tasks():
    """Load tasks from file if it exists"""
    global tasks, task_id_counter
    try:
        if os.path.exists("tasks.json"):
            with open("tasks.json", "r") as f:
                tasks = json.load(f)
                # Update counter
                if tasks:
                    task_id_counter = max(t["id"] for t in tasks) + 1
            print("✓ Previous tasks loaded!")
    except Exception as e:
        print(f"Error loading tasks: {e}")

def add_task(title):
    """Add a new task"""
    global task_id_counter
    task = {
        "id": task_id_counter,
        "title": title,
        "completed": False
    }
    tasks.append(task)
    task_id_counter += 1
    print(f"✓ Task added! ID: {task['id']}")

Project - Main Loop

Putting It All Together

def main():
    """Main application loop"""
    load_tasks()  # Load previous tasks if they exist

    while True:
        display_menu()

        try:
            choice = input("\nEnter your choice (1-5): ").strip()

            if choice == "1":
                title = input("Enter task description: ").strip()
                if title:
                    add_task(title)
                else:
                    print("Task cannot be empty!")
            elif choice == "2":
                display_tasks()
            elif choice == "3":
                display_tasks()
                task_id = int(input("Enter task ID to mark complete: "))
                mark_complete(task_id)
            elif choice == "4":
                display_tasks()
                task_id = int(input("Enter task ID to delete: "))
                delete_task(task_id)
            elif choice == "5":
                save_tasks()
                print("✓ Goodbye!")
                break
            else:
                print("Invalid choice! Please try again.")

        except ValueError:
            print("Invalid input! Please enter a number.")
        except Exception as e:
            print(f"An error occurred: {e}")

if __name__ == "__main__":
    main()
