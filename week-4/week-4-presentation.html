What is Django?

Django: A Python Web Framework

"The web framework for perfectionists with deadlines"
  • Written in Python
  • Batteries included (everything you need)
  • Fast development
  • Secure by default
  • Scalable
  • Great documentation

Used by: Instagram, Pinterest, Mozilla, Spotify, NASA

Web Framework - Why?

Without Framework:

  • ❌ Handle HTTP manually
  • ❌ Parse URLs yourself
  • ❌ Generate HTML by hand
  • ❌ Manage database
  • ❌ Handle security
  • ❌ Write everything

With Django:

  • HTTP handled
  • URL routing built-in
  • Template engine
  • ORM included
  • Security features
  • Focus on your app

Framework = Foundation + Tools

MTV Pattern

Django uses MTV:

M = Model   (Database layer)      ← Week 5
T = Template (Presentation layer)  ← Week 4
V = View    (Business logic)       ← Week 4

Similar to MVC (Model-View-Controller)

User Request → URL → View → Template → Response

Installing Django

Step 1: Create Virtual Environment

python -m venv venv
venv\Scripts\activate  (Windows)
source venv/bin/activate  (Mac/Linux)

Step 2: Install Django

pip install django

Step 3: Verify

django-admin --version

That's it! Django is ready.

Create Django Project

# Create project
django-admin startproject mysite

# Navigate in
cd mysite

# Run server
python manage.py runserver

# Visit http://localhost:8000
# See Django welcome page!
Trainer Note: Live demo - show the Django welcome page in browser

mysite/

Project vs App:

  • Project: The entire website
  • App: A component (blog, users, store)
  • One project can have many apps

What is a View?

A View is a Python function that:

  1. Receives a request
  2. Processes it (fetch data, logic)
  3. Returns a response (HTML, JSON, etc.)

Example:

from django.http import HttpResponse

def home(request):
    return HttpResponse("Hello, Django!")

URL Routing

URLs connect web addresses to views:

# core/urls.py
from django.urls import path
from . import views

urlpatterns = [
    path('', views.home, name='home'),
    path('about/', views.about, name='about'),
]
# mysite/urls.py
from django.urls import include

urlpatterns = [
    path('', include('core.urls')),
]

Request-Response Cycle

User visits URL
      ↓
Browser sends HTTP request
      ↓
Django checks urls.py
      ↓
Finds matching URL pattern
      ↓
Calls view function
      ↓
View returns HTTP response
      ↓
Browser displays page

Django Templates

Templates = HTML + Django Template Language

Why templates?

  • ❌ return HttpResponse("HTML as string") # Hard to maintain
  • ✓ return render(request, 'template.html') # Separate & clean

Template features:

  • Variables: {{ name }}
  • Logic: {% if %}, {% for %}
  • Inheritance: {% extends %}
  • Include: {% include %}

Template Variables

In views.py

def home(request):
    context = {
        'name': 'Ahmed',
        'age': 25,
        'skills': ['Python', 'Django', 'HTML']
    }
    return render(request, 'home.html', context)

In home.html

<!-- In home.html -->
<h1>Welcome, {{ name }}!</h1>
<p>Age: {{ age }}</p>

Template Tags - If/Else

{% if user_logged_in %}
    <p>Welcome back, {{ username }}!</p>
{% else %}
    <p>Please log in.</p>
{% endif %}

Template tags use {% ... %}
Conditions work like Python

Template Tags - For Loops

<h2>My Skills:</h2>
<ul>
    {% for skill in skills %}
        <li>{{ skill }}</li>
    {% empty %}
        <li>No skills listed</li>
    {% endfor %}
</ul>

Loops through lists, displays each item
{% empty %} handles empty lists

Template Inheritance

base.html

<html>
<head>
    <title>{% block title %}My Site{% endblock %}</title>
</head>
<body>
    <header>
        <h1>My Website</h1>
    </header>
    <nav><!-- navigation --></nav>

    {% block content %}
    {% endblock %}

    <footer>© 2025</footer>
</body>
</html>

Extending Base Template

home.html

<!-- home.html -->
{% extends 'base.html' %}

{% block title %}Home - My Site{% endblock %}

{% block content %}
    <h2>Welcome!</h2>
    <p>This is the home page.</p>
{% endblock %}

Benefits:

  • No code duplication
  • Consistent design
  • Easy to maintain

Static Files

Static Files = Files that don't change:

  • CSS stylesheets
  • JavaScript files
  • Images
  • Fonts

Setup:

core/
└── static/
    └── core/
        ├── css/
        │   └── style.css
        └── images/
            └── logo.png

Using Static Files

{% load static %}  <!-- At top of template -->

<!DOCTYPE html>
<html>
<head>
    <link rel="stylesheet" href="{% static 'core/css/style.css' %}">
</head>
<body>
    <img src="{% static 'core/images/logo.png' %}" alt="Logo">
</body>
</html>

{% static %} generates correct URL
Always use {% load static %} first
